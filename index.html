Push new rows
Detects rows not yet in ClickUp using composite key (Order ID + Item name) and pushes only new ones — never duplicates.
⬇️
Pull updates
Fetches ClickUp task changes back to Sheet. Pre-flight check blocks tasks missing Order ID before any rows are written.
🔄
Update existing
Overwrite ClickUp fields from Sheet, or Sheet fields from ClickUp — two separate manual buttons for full control.
⏹️
Stop & resume
Pause a large push mid-batch and resume exactly where it stopped. Progress persists across sidebar reloads.
🔧
Fill missing fields
Scans ClickUp tasks for empty fields and fills them from Sheet using Order ID as the match key. Gated — requires list + mapping.
🔍
Duplicate detection
Finds tasks sharing same Order ID + Item name. Shows grouped list with per-task delete buttons and confirmation prompts.
Technical highlights
Retry logic with exponential backoff
Every ClickUp API call retries up to 3 times with 1s / 2s / 3s delays. Per-page retry prevents failure on page 14+ of large task fetches (2,000+ tasks across 20+ pages).
Composite deduplication key
Uses OrderID|ItemName as the composite key — same order ID with different items (bundle or split orders) are treated as separate valid rows, not duplicates.
Config validation with clear errors
Before every sync operation, validates that Task Name and Order ID fields are mapped. Shows specific error messages in the sidebar banner linking directly to Settings.
Two-step dashboard loading
Sheet counts load instantly (~0.5s by reading local data). ClickUp counts fetch in the background (~10–15s via API). Sidebar is usable immediately on open.
Live progress polling
During push, sidebar polls every 2 seconds for live Processed / Pushed / Errors counts with a progress bar and ETA estimate. Progress saved to Script Properties every 5 rows.
Multi-list support
Supports multiple ClickUp lists with different custom field IDs for the same logical field (e.g. Order ID). Each list's field IDs are mapped independently.
Sidebar UI — 3-tab Google Sheets panel
🔄 Sync tab
Live dashboard showing Sheet vs ClickUp row counts, unsynced items with Order ID preview, last sync activity grid (4 directions), push/pull buttons with pre-flight checks, stop/resume controls, live syncing counter.
🛠 Tools tab
Fill Missing Fields (gated — shows error if list or mapping not configured), Duplicate Detection with per-task delete, prerequisite validation with links to Settings, delete confirmation dialogs.
⚙️ Settings tab
Per-tab ClickUp list picker (Space → Folder → List with skip-folder option), dynamic field mapping table with add/remove rows, saved independently per sheet tab using Script Properties.
Build timeline
Core sync engine Foundation
Built push/pull with ClickUp REST API. Field mapping, status map, dropdown option ID resolution for Brand, Item, and Platform fields. Two-step task creation (name+status first, then fields individually).
Batch processing & reliability
500-row batches with stop/resume. Retry logic with exponential backoff. 500ms page delays. Per-page retry to handle 2,000+ task lists without hitting timeout limits.
Smart deduplication Key feature
Composite key (Order ID + Item name) prevents duplicate tasks while correctly allowing same order ID with different items. Applied to push, update, and duplicate scanning consistently.
Multi-tab support
Each sheet tab gets its own ClickUp list and field mapping, saved to Script Properties. Supports (1) Ecom Orders and ECOM BATCH ORDERS independently with different field structures.
Pre-flight validation
Push checks Sheet rows for missing Order ID before syncing. Pull checks ClickUp tasks for missing Order ID. Both show flagged items with fix-or-skip options. Config validates required field mappings on every operation.
Performance optimization Final polish
Two-step loading (sheet instant, ClickUp background). Live progress counter polling every 2s. getSheetCounts returns config status in same call to eliminate extra round trips. Nothing auto-loads on sidebar open.
